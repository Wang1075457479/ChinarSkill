# Trello任务创建Skill

## 功能概述

自动解析自然语言创建规范化的Trello任务卡片，包括：
- 智能任务拆解（一句话→标题+描述+清单）
- 自动成员分配
- 智能截止日期（当月最后一个工作日）
- 默认放入"Not Start"列表

## 触发方式

当用户说以下内容时激活此Skill：
- `给"张三"创建一个xxx的任务`
- `创建任务`
- `创建Trello任务`
- `帮我建个任务`
- `安排任务给XX`
- `新建任务`

## 任务规范

### 标题格式
- 格式：`【分类】-任务标题`
- 分类：从以下选择（≤5字）：开发、优化、安全、规则、设计、测试、文档、重构
- 限制：总体不超过50个汉字（完整显示任务内容）
- 示例：`【开发】-用户登录模块功能开发`、`【优化】-系统性能提升与卡顿问题修复`
- **XMind规则**：每个画布对应一个任务，标题取画布完整名称

### 描述格式
- 用户原始需求的完整描述
- 可以是一句话或多句话
- 保持原意，不添加额外内容
- **不要包含附件路径**，附件单独上传
- **XMind规则**：自动生成≤100字的内容摘要，包含画布名称和核心节点

### 清单格式
- 将任务拆解为具体执行步骤
- 每个步骤用数字前缀：`1、xxx`、`2、xxx`、`3、xxx`
- 步骤要具体、可执行、可验证，禁止无意义名称（如"大小"、"方式"、"分类"等）
- 多级节点自动组合成完整任务描述，例如：父节点"数据库"+子节点"新增名称字段" → "数据库：新增名称字段"
- 限制：每个任务的清单最多50条
- **XMind规则**：
  - 画布下每个分支自动总结为一条可执行任务
  - 保留层级结构，多级节点组合生成有意义的步骤描述
  - 自动过滤无意义的节点名称，合并成可阅读的任务
  - 超过50条自动截断

### 默认配置
- 默认列表：`Not Start`
- 截止日期：当月最后一个工作日
- 成员：从看板成员中动态匹配
- **默认规则**：用户未明确说明创建多个任务时，默认只创建一个任务卡片

## 使用流程

### 手动创建任务
1. **解析用户输入**
   - 提取任务内容
   - 提取被分配人（如果有）
   - 提取附件（如果有）
   - 支持XMind/Excel/文字等多源数据解析

2. **智能任务拆解**
   - 生成符合规范的标题（【分类】-任务标题，≤15字）
   - 自动分析任务分类（开发/优化/安全/规则/设计/测试/文档/重构）
   - 保留原始描述（≤100字摘要）
   - 拆解为带数字前缀的清单（≤50条，自动组合层级节点，过滤无意义名称）
   - XMind规则：每个画布对应一个任务卡片，分支自动转为清单项

3. **获取看板数据**
   - 调用Trello API获取看板成员列表
   - 匹配被分配人姓名到Trello用户ID

4. **计算截止日期**
   - 获取当月最后一天
   - 如果是周末，向前调整到最后一个工作日

5. **创建任务卡片**
   - 调用Trello API创建卡片
   - 设置标题、描述、清单
   - 添加成员、截止日期
   - 上传附件（如果有）

### 自动整理功能
1. **自动执行**
   - 每3小时自动扫描Trello看板所有任务
   - 按照技能规范自动整理

2. **整理规则**
   - 标准化标题格式，自动识别分类，限制≤15字
   - 没有清单的任务自动根据描述生成可执行清单项（≤50条）
   - 不修改原有任务含义，不删除附件、评论等原有内容
   - 自动过滤无意义名称，生成规范的任务描述

3. **通知机制**
   - 每次整理完成后自动发送飞书通知
   - 通知包含整理结果和变更明细

## API配置

使用前需要配置：
- `TRELLO_API_KEY`：Trello API密钥
- `TRELLO_TOKEN`：Trello访问令牌
- `TRELLO_BOARD_ID`：目标看板ID

配置方式：创建`references/config.json`文件
