# Trello任务创建技能优化说明

## 优化完成时间
2026-03-11

## 优化内容

### 1. 标题标准化功能增强 ✅
**解决问题：** 原标题格式混乱，包含多余符号、数字前缀、嵌套括号等问题
- 自动识别并清理所有不符合规范的标题格式
- 支持处理各种错误格式：`【设计】-【BUG】-网络波动` → `【BUG】-网络波动`
- 支持处理带数字前缀的标题：`1、登录页面开发` → `【开发】-登录页面`
- 支持处理带括号的标题：`[BUG] 网络请求超时` → `【BUG】-网络请求超时`
- 支持处理嵌套括号和多余符号：`【设计】-长期搁置-【拼搭方式-多轴联动】` → `【设计】-拼搭方式-多轴联动`
- 严格控制标题长度≤30字，超过自动截断并添加省略号
- 自动识别任务分类（开发/优化/BUG/设计等9种分类）

### 2. 清单标准化功能增强 ✅
**解决问题：** 清单序号格式混乱、重复、不足3条等问题
- 统一使用 `1、2、3` 作为序号前缀，彻底杜绝格式混乱
- 自动清理所有原有序号：`一、xxx`、`2. xxx`、`3- xxx`、`第x步 xxx` 等
- 自动去重，避免重复清单项
- 不足3条时自动补充到3条，确保每个任务至少有3个执行步骤
- 最多支持50条清单项，超过自动截断
- 支持混合格式输入，输出完全统一规范

### 3. XMind文件支持 ✅
**新增功能：** 支持直接解析XMind文件创建任务
- 每个XMind画布对应生成一个Trello任务卡片
- 自动提取画布名称作为任务标题，自动标准化
- 自动解析画布下的所有分支节点作为清单项
- 自动整理层级结构，生成清晰的执行步骤
- 自动标准化清单格式，统一序号
- 支持XMind 2020+版本文件

### 4. 自动整理功能优化 ✅
**解决问题：** 定时任务整理时破坏原有标题和内容的问题
- 保留原有任务的所有描述、附件、评论、关联信息，只修改标题和补充清单
- 标题修改前会严格检查是否已经符合规范，符合规范的不做任何修改
- 已有清单的任务会自动标准化清单项格式，不删除原有内容
- 没有清单的任务自动根据描述生成3条以上清单项
- 支持批量整理整个看板的所有任务，无侵入式修改

## 格式规范

### 标题格式规范（强制）
```
【分类】-任务简述
```
- 分类：开发、优化、安全、规则、设计、测试、文档、重构、BUG（9种可选）
- 长度：≤30字（包含符号）
- 示例：`【开发】-用户登录模块`、`【BUG】-支付页面崩溃修复`

### 清单格式规范（强制）
```
1、[动词]开头的执行步骤
2、[动词]开头的执行步骤
3、[动词]开头的执行步骤
...
```
- 统一使用阿拉伯数字加顿号作为前缀
- 每项以动词开头，明确可执行
- 最少3条，最多50条

## 使用方法

### 1. 普通任务创建
```python
from trello_task_creator import TrelloTaskCreator

creator = TrelloTaskCreator()
task_data = {
    "title": "登录页面开发",
    "description": "实现用户登录功能，支持手机号和邮箱登录",
    "checklist": ["UI设计", "表单验证", "API对接", "测试"],
    "assignee": "王晨阳"
}
card = creator.create_task(task_data)
```

### 2. 从XMind创建任务
```python
from trello_task_creator import TrelloTaskCreator

creator = TrelloTaskCreator()
# 每个画布生成一个任务卡片
cards = creator.create_task_from_xmind("path/to/your/file.xmind", assignee="王晨阳")
```

### 3. 手动运行自动整理
```bash
cd scripts
python trello_auto_organizer.py
```

## 注意事项
1. 定时任务每3小时自动运行一次，自动整理看板任务
2. 整理时不会删除任何原有内容，只会补充和标准化
3. 标题修改后会保留原标题的核心含义，不会改变任务本意
4. XMind导入时每个画布只生成一个卡片，不会创建多个卡片
