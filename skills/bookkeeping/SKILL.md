# 飞书账单管家 Skill

## 核心原则 (Core Rules)

### 1. 绝对服从分类定义
严禁按字面意思盲目分类，必须遵循以下【分类路由规则】：

**👉 目标表：人 (个人日常开销)**
- 定义：属于个人生活、吃喝玩乐、个人出行、保险的消费
- 典型包含：
  - 餐饮、外卖、电影、购物
  - 打车/滴滴/高德打车/货拉拉（注意：打车属于个人消费，绝对不能放入"车"）
  - 游戏充值（如：列车补给凭证、月卡、Steam等）
  - 保险/医保（众安保险、好医保、健康险等，统一归入人表）
  - 个人医疗、话费充值

**👉 目标表：车 (车辆专属开销)**
- 定义：仅限车辆实体本身的硬性维护和运行开支
- 典型包含：
  - 加油、充电、洗车、车辆维修、保养
  - ETC通行费、车配件（如：脚垫、行车记录仪）
  - 车贷定金、车险（注：打车绝不在内）

**👉 目标表：家 (家庭与公共开销)**
- 定义：家庭共同使用的物品、服务、以及家庭资产防护
- 典型包含：
  - 水电煤气费、买菜、家具家电、生活日用品
  - 装修、建材、家居用品
  - 物业费、房租

### 2. 保持状态静默
所有写入的数据，其【状态】字段（复选框）必须保持为 **false**（未勾选），留给用户后续人工审核，严禁自作主张勾选！

### 3. 字段完整性
- 【时间】必须有值，使用UTC时间戳（北京时间-8小时）
- 【品牌】尽最大努力提取，不在预定义列表中的品牌留空或放入备注
- 【备注】包含交易描述、退款信息等

### 4. 严格执行过滤
满足过滤条件的数据直接抛弃，不予写入。

## 工作流程 (Workflow)

### Step 1: 数据解析与初步过滤

当收到用户输入的数据（Excel、CSV、图片），立即执行以下过滤：

**【规则 A - 小额过滤】**
消费金额 **低于 9 元（< 9.00）** 的账单数据，直接丢弃，不予记录。

**【规则 B - 退款处理】**
识别为"退款"的账单，需与原消费记录进行抵消比对：
- 全额退款，且原账单未录入：双向忽略
- 部分退款：计算【实际支出金额】后再录入，并在【备注】中注明"包含退款金额 xx 元"

**【规则 C - 内部转账过滤】**
包含以下关键词的记录不计入：
- 余额宝、花呗自动还款、信用卡还款
- 余利宝、网商银行、提现
- 收益发放、转入、转出、充值、零钱通

### Step 2: 智能分类路由

根据交易实质划分到三个数据表：

```python
# 分类逻辑
def classify(name, description):
    text = name + description
    
    # 保险/医保 -> 人
    if '保险' in name or '医保' in name or '健康' in name:
        return '人'
    
    # 车相关 -> 车
    car_keywords = ['加油', '充电', '洗车', '维修', '保养', 'ETC', '车险', '车贷', '车辆', '小米汽车']
    if any(kw in text for kw in car_keywords):
        return '车'
    
    # 家相关 -> 家
    home_keywords = ['装修', '家居', '建材', '家具', '家电', '灯具', '插座', '开关', '网线']
    if any(kw in text for kw in home_keywords):
        return '家'
    
    # 其他 -> 人
    return '人'
```

### Step 3: 字段精准提取

**【名称】**：精简消费名称（如"李阳羊肉"、"公牛插座"）

**【费用】**：提取纯数字（保留两位小数）

**【品牌】**：必须执行品牌实体识别
- 从名称或描述中剥离品牌词
- 例如："小米电磁炉"->品牌为【小米】；"公牛插座"->品牌为【公牛】；"淘宝-xxx"->品牌为【淘宝】
- 如果品牌不在预定义列表中，留空不填

**【最后更新时间 / 消费时间】**：绝对不可为空
- 优先级1：从账单明细中提取精确的交易时间
- 优先级2：北京时间转换为UTC时间戳（减去8小时）

**【状态】**：必须为 **false**（未勾选）

**【备注】**：将账单的附加信息填入

### Step 4: 执行写入 (API Execution)

1. 检查上述步骤是否全部合规
2. 获取飞书 tenant_access_token
3. 调用飞书多维表格 API 进行批量写入
4. 使用 时间+名称+费用 作为唯一键，避免重复导入
5. 输出执行报告
